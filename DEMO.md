# DEMO.md — 现场演示手册

面向面试现场的精确演示步骤，每步附一句口播话术。建议演示前确认 8000 / 5500 端口未被占用。

## 0. 演示前准备（30 秒）
- 确认已安装 Python 3.9+。
- 关闭可能占用 8000/5500 的旧进程。
- 口播：「这是一套无人机森林火情巡检与预警平台，后端 FastAPI、前端纯静态页、再加一个无人机模拟端，全部本地一键启动。」

## 1. 一键启动（双击 start.vbs）
- 双击项目根目录的 `start.vbs`（推荐，无黑窗静默启动），它会自动：创建项目本地 `.venv` 虚拟环境 → 在 `.venv` 内装依赖 → 后台起后端(8000) → 起前端静态服务(5500) → 起模拟端 → 待端口就绪后自动打开浏览器到 `http://localhost:5500`，全程不弹出黑色终端窗口。
- 排错可用 `start_debug.bat`（三个可见窗口看各服务实时输出），停止用 `stop.bat`。
- 口播：「我双击这个启动脚本，它会在后台自动拉起后端、前端和无人机模拟端，并打开监控页面，整个过程没有黑窗，像成品一样双击即用。」

## 2. 无人机列表页（看在线 / 电量）
- 浏览器停在 `http://localhost:5500`，默认在无人机列表页。
- 指出：5 架无人机、状态在线/巡航、电量百分比、最后心跳时间；页面每 2 秒自动刷新，数字在动。
- 口播：「现在 5 架无人机都已自动注册上线，这里实时显示它们的状态、电量和最后心跳，页面每两秒自动刷新。」

## 3. 巡检地图页（看实时位置 / 轨迹）
- 切到巡检地图页。
- 指出：地图上无人机标记在缓慢移动，身后拖出历史巡检轨迹线；疑似火点会以红色高亮。
- 可切换「标准 / 卫星 / 示意」三种底图，其中示意地图不依赖网络、可离线演示。
- 口播：「地图上能看到每架无人机的实时位置和巡检轨迹，一旦识别到疑似火点，会在对应位置高亮标记；底图支持标准、卫星和示意三种，示意地图不依赖网络，断网也能完整演示。」

## 4. 手动触发高危火情（可控演示核心）
- 在项目根目录运行（依赖在 `.venv` 里，用 `.venv` 的 python 避免全局 python 报缺 requests）：
  ```bash
  .venv\Scripts\python.exe scripts\demo_trigger.py --level high
  ```
- 脚本会自动选一架在线无人机，上报一条高危数据，并打印返回的 `alert_id` 和等级。
- 口播：「为了演示告警链路，我现在手动触发一个高危火情——脚本向后端上报一条高温高烟雾数据，后端立刻按规则生成了一条 high 级告警。」
- 可选：`--level medium` / `--level low` 演示不同等级；`--drone sim_02` 指定无人机。

## 5. 告警中心页（确认 / 忽略）
- 切回浏览器，进入告警中心页（约 2 秒内刷新出刚才的 high 告警）。
- 点开告警看详情（编号、无人机、位置、等级、原因、时间、状态）。
- 告警原因为中文且带具体数值与阈值（如「温度过高（95℃，阈值80℃）」），详情含「现场图像/识别快照」链接。
- 点「确认」→ 状态变已确认；或点「忽略」→ 状态变已忽略；顶部未处理计数相应减少。
- 口播：「刚触发的高危告警已经出现在告警中心，点开能看到完整信息——触发原因是中文加具体数值和阈值，还带现场图像识别快照链接。值班员可以确认或忽略，状态会实时回写。」

## 6. 收尾
- 口播：「整套链路：无人机上报 → 后端规则判定生成告警 → 前端实时展示与处置，全部跑通。数据存内存、零外部依赖，一个脚本即可复现。」
- 提示评委：数据为内存存储，重启后端会清空，演示期间请勿重启后端。
- 演示结束双击 `stop.bat` 释放 8000 / 5500 端口。

## 备用：纯接口演示（前端不可用时）
- 后端接口文档：`http://localhost:8000/docs` 可直接在线调用全部接口。
- 验证模拟端：`GET http://localhost:8000/api/drones`（多刷几次看数据在变）、`GET http://localhost:8000/api/drones/sim_01/track`（看轨迹增长）。
